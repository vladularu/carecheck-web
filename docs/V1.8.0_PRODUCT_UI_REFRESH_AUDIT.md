# v1.8.0 Product UI Refresh Audit

Datum: 2026-07-15

## Ziel

v1.8.0 bereitet eine sauberere und primaer iPhone-orientierte Produktoberflaeche vor. Der Refresh ist bewusst als eigener Zwischenschritt vor Sync/Auth eingeordnet, damit v2.0 fachlich auf Benutzerkonten und Synchronisierung fokussiert bleiben kann.

## Umgesetzt

- Branch `feature/v1.8.0-product-ui-refresh` angelegt.
- Bottom-Navigation fuer iPhone neu geordnet:
  - Hauptbereiche: Start, Plan, Kalender, Check
  - Nebenbereiche im Mehr-Menue: Bericht, Gehalt, Jahr, Fairness, Profil
- Klassische Desktop-Navigation bleibt vollstaendig sichtbar.
- App-Shell, Seitenkoepfe, Karten, Buttons, Formulare und Planerbereiche visuell beruhigt.
- Dashboard-Status von Emoji-Markern auf sachlichen Statuspunkt umgestellt.
- Geschuetzte Referenz `CareCheck Vision` nach Freigabe inspiziert und Look-and-feel uebernommen:
  - Paper/Mint/Ink-Farbwelt
  - ruhige Monatscockpit-Karten
  - pillige Status- und Tab-Zustaende
  - iPhone-nahe Bottom-Navigation
- Roadmap ergaenzt:
  - `v1.8.0 Product UI Refresh`
  - `v1.9.0 Sync-Vorbereitung und Datenmodell`
  - `v2.0 Synchronisierung und Mehrgeraetebetrieb`

## Fachliche Abgrenzung

- Keine Aenderung an Statistik-, Tarif-, Export- oder Compliance-Berechnungslogik.
- Keine Aenderung am Storage- oder Backup-Datenmodell.
- Keine Aenderung an ArbZG- oder TVoeD-P-Pruefregeln.

## Validierung

- ESLint erfolgreich.
- Vitest erfolgreich: 30 Testdateien, 177 Tests.
- Produktions-Build erfolgreich.
- Desktop-DOM-Pruefung ohne horizontales Layout-Overflow.

## Offen

- Demo-Referenzen oder Screenshots abgleichen, sobald vorhanden.
- Mobile Sichtpruefung auf echtem iPhone oder verlaesslichem iPhone-Viewport nachziehen. Der In-App-Browser hat den gesetzten iPhone-Viewport in dieser Sitzung nicht uebernommen.
- Nach finaler visueller Abnahme: Merge nach `main`, Release-Tag und GitHub Release fuer `v1.8.0`.
